# Альфа-CBDC · MVP трансграничных расчётов в цифровом рубле

MVP комплексного банковского продукта для клиентов-юридических лиц Альфа-Банка
по проведению трансграничных платежей в **цифровом рубле** через платформу
Банка России с последующей атомарной конвертацией в **цифровой юань (e-CNY)**,
**цифровой дирхам (e-AED, проект Aber/mBridge)** или **цифровой белорусский
рубль (e-BYN)**.

> Проект демонстрирует все шесть функциональных модулей сервиса, описанных в
> продуктовой документации: открытие кошелька цифрового рубля для юрлица,
> пополнение/вывод, трансграничный платёж через интероперабельный коридор,
> смарт-контракт «Безопасная сделка», валютный контроль по Инструкции
> Банка России № 181-И и программу лояльности.

## ⚡ Быстрый запуск

### Linux / macOS
```bash
git clone <your-repo>.git alfa-cbdc
cd alfa-cbdc
./scripts/run.sh
```

### Windows
```bat
git clone <your-repo>.git alfa-cbdc
cd alfa-cbdc
scripts\run.bat
```

### Вручную
```bash
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

Откройте: **http://127.0.0.1:8000**

API-документация (Swagger UI): **http://127.0.0.1:8000/docs**

### Через Docker
```bash
docker build -t alfa-cbdc .
docker run -p 8000:8000 alfa-cbdc
```

При первом запуске автоматически создаются БД SQLite (`alfa_cbdc.db`) и
демонстрационные данные: три клиента разных сегментов (малый/средний/крупный),
по контрагенту в каждом коридоре (CN/AE/BY) и зарегистрированные ВЭД-контракты.

## 🧩 Реализованная бизнес-логика

| # | Модуль | Реализация |
|---|--------|------------|
| 1 | **Кошелёк цифрового рубля для юрлица** | `wallet_service.open_wallet_for_client` — открывается только после KYC по 115-ФЗ; выдаётся ID кошелька на платформе Банка России. |
| 2 | **Пополнение / вывод** | `top_up` / `withdraw` — соотношение 1:1, без комиссии за пополнение. |
| 3 | **Трансграничный платёж** | `payment_service.initiate_payment(payment_type="instant")` — расчёт котировки → комплаенс → **атомарное списание drub + зачисление ЦВЦБ-партнёра** в одной транзакции (CBR-TX + PBOC-TX/CBUAE-TX/NBRB-TX). |
| 4 | **Смарт-контракт «Безопасная сделка»** | `payment_type="smart_contract"` — сумма блокируется на эскроу-кошельке; есть три условия (электронная транспортная накладная, ФТС, приёмка груза). При выполнении всех условий — автоисполнение, при истечении дедлайна или ручном refund — возврат. |
| 5 | **Валютный контроль (181-И)** | `currency_control_service` — присваивается УНК, по каждой операции формируется СПД, пакет передаётся в Банк России и ФНС России. |
| 6 | **Программа лояльности** | KPI `loyalty_clients` на дашборде — число клиентов с ≥5 операциями за 30 дней. |

### Тарифная сетка (см. `app/config.py`)
| Сегмент | Размер сделки | Комиссия |
|---|---|---|
| Малый | 5–20 млн ₽ | 2.0% |
| Средний | 20–250 млн ₽ | 1.0% |
| Крупный | >250 млн ₽ | 0.5% |

### Коридоры ЦВЦБ
| Код | Юрисдикция | Валюта | Кросс-курс (демо) | Доля |
|---|---|---|---|---|
| CN | 🇨🇳 КНР (PBoC) | e-CNY | 1 e-CNY = 12.50 цифр. ₽ | 60% |
| AE | 🇦🇪 ОАЭ (CBUAE, mBridge) | e-AED | 1 e-AED = 24.30 цифр. ₽ | 25% |
| BY | 🇧🇾 Беларусь (НБ РБ) | e-BYN | 1 e-BYN = 27.80 цифр. ₽ | 15% |

## 🖱️ Пользовательский путь

1. **Дашборд** — посмотреть сводные KPI и подключённые коридоры.
2. **Клиенты & кошельки** — выбрать демо-клиента в верхней панели; пополнить/вывести цифровые рубли.
3. **Контрагенты & контракты** — добавить иностранного контрагента или новый контракт (УНК присвоится автоматически).
4. **Трансграничные платежи** — выбрать контракт, коридор и сумму. Котировка пересчитывается в реальном времени. Отправить как мгновенный платёж или как смарт-контракт.
5. **Смарт-контракты** — последовательно подтверждать условия (накладная → ФТС → приёмка). После третьего условия платёж исполняется автоматически.
6. **Валютный контроль** — посмотреть автоматически сформированный пакет документов (УНК, СПД, payload для ЦБ и ФНС).
7. **Журнал аудита** — все операции платформы с временными метками.

## 📚 Юридические основания

- **ФЗ № 340-ФЗ** — о внесении изменений в части введения цифрового рубля
- **ФЗ № 115-ФЗ** — ПОД/ФТ, KYC/KYB
- **Инструкция Банка России № 181-И** — порядок представления резидентами и нерезидентами уполномоченным банкам подтверждающих документов и информации при осуществлении валютных операций

## ⚠️ Дисклеймер

Это **исследовательский MVP**. Кросс-курсы, идентификаторы транзакций
(`CBR-TX-…`, `PBOC-TX-…`), интероперабельные шлюзы между платформами ЦВЦБ
и интеграции с ФТС/ФНС России — **демонстрационные**. На момент создания
проекта (2026) интероперабельные коридоры Банка России с ЦБ КНР, ОАЭ и
Беларуси находятся в разной стадии готовности; цель MVP — показать
архитектуру и пользовательский опыт продукта.
